Trofiro is built around a small set of well-defined concepts that work together to form a cohesive data model. Before you start creating resources, building workflows, or connecting third-party services, it helps to understand what these building blocks are and how they relate to each other. This page introduces each concept at a high level so the rest of the documentation clicks into place.

## Workspace

A workspace is the top-level container for everything your team does in Trofiro. It holds your resources, workflows, integrations, settings, and member accounts under a single organizational unit. Every Trofiro account belongs to at least one workspace, and you can create multiple workspaces to separate projects, clients, or environments such as production and staging. Permissions, billing, and API keys are all scoped to the workspace level, so each workspace operates independently.

## Resource

A resource is the primary unit of work in Trofiro — it represents anything you create, track, or manage inside your workspace. Resources are flexible by design: they carry a standard set of fields such as a name, status, and timestamps, and you can extend them with custom fields to match your team's specific needs. Everything else in Trofiro — workflows, tags, relationships, API calls — ultimately operates on resources.

## Workflow

A workflow is an automated sequence of actions that Trofiro executes in response to events on your resources. Instead of manually following up every time something changes, you define a trigger (the event that starts the workflow), an optional condition (a filter that must pass), and one or more actions (the tasks Trofiro performs). Workflows run in the background so your team can focus on higher-value work. See the [Workflows](/concepts/workflows) page for a full breakdown of available triggers and actions.

## Integration

An integration is a connection between Trofiro and a third-party service such as Slack, GitHub, or Zapier. Once an integration is configured, it can be used as an action inside workflows — for example, posting a Slack message when a resource reaches a certain status — or as a data source that syncs information into your workspace. Integrations are authorized with OAuth or API tokens, and you can manage them from the **Integrations** section of your workspace settings.

## Webhook

A webhook is an HTTP callback that Trofiro sends to a URL you specify whenever a particular event occurs. Unlike polling the API on a schedule, webhooks let your server react to changes in real time. You register a webhook endpoint, choose which events it should receive, and Trofiro delivers a signed JSON payload to that URL each time a matching event fires. Webhooks are commonly used to sync data with external systems, trigger CI/CD pipelines, or build custom notification flows.

## API Key

An API key is a credential that authenticates programmatic access to your Trofiro workspace via the REST API. You generate API keys from **Settings → API Keys**, and each key can be scoped to specific permissions so you can grant the minimum access a particular integration or script needs. Treat API keys like passwords: store them in environment variables or a secrets manager, and rotate them immediately if you suspect a key has been compromised.

<Warning>
  Never commit API keys to source control. Use environment variables or a dedicated secrets manager to keep your credentials safe.
</Warning>
