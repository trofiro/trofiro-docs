Workflows are Trofiro's built-in automation engine. Instead of manually reacting to every change in your workspace, you define the rules once and let Trofiro handle the follow-up automatically. Whether you need to notify your team when a high-priority resource is created, archive stale records on a schedule, or push data to an external system the moment something changes, workflows give you the flexibility to build those processes without writing custom code.

## How Workflows Work

Every workflow follows a three-step model: **trigger → condition → action**.

- **Trigger** — An event that starts the workflow, such as a resource being created or a scheduled time arriving. Trofiro listens for the trigger continuously so your automation is always on.
- **Condition** *(optional)* — A filter that must evaluate to `true` before the workflow proceeds. Conditions let you narrow scope — for example, only run when `status` equals `active` or when a tag contains `urgent`. If the condition is not met, Trofiro skips the workflow for that event without raising an error.
- **Action** — The task (or series of tasks) Trofiro performs when the trigger fires and the condition passes. Actions can send notifications, manipulate resources, call external services, or execute scripts.

This linear model keeps workflows easy to reason about while still covering a wide range of real-world automation needs.

## Creating a Workflow

<Steps>
  <Step title="Open the Workflows section">
    In the left sidebar, click **Workflows**. You'll see a list of all existing workflows in your workspace along with their current status (active or inactive).
  </Step>
  <Step title="Start a new workflow">
    Click **New Workflow** in the top-right corner. Give your workflow a descriptive name so teammates can understand its purpose at a glance.
  </Step>
  <Step title="Choose a trigger">
    Select the event that will start the workflow. Available triggers include **Resource Created**, **Resource Updated**, **Resource Deleted**, and **Scheduled**. Each trigger type exposes additional configuration — for example, the Scheduled trigger lets you specify a cron expression.
  </Step>
  <Step title="Add a condition (optional)">
    Click **Add Condition** to filter which events actually proceed. You can match on any resource field — for instance, only continue when `tags` includes `priority:high` or when `status` changes to `archived`. Leave this step empty if you want the workflow to run for every matching trigger event.
  </Step>
  <Step title="Add one or more actions">
    Click **Add Action** and choose what Trofiro should do. You can chain multiple actions together — they execute in the order you define them. Examples include **Send notification**, **Create resource**, and **Call webhook**. Each action type has its own configuration panel where you can reference trigger data using template variables such as `{{resource.name}}`.
  </Step>
  <Step title="Save and activate">
    Click **Save**, then toggle the workflow to **Active**. Trofiro will begin listening for the trigger immediately. You can deactivate a workflow at any time without deleting it, which is useful for pausing automation during maintenance windows.
  </Step>
</Steps>

## Workflow Triggers

The following triggers are available when building a workflow:

| Trigger | Description |
|---|---|
| Resource Created | Fires when a new resource is created in the workspace. |
| Resource Updated | Fires when any field on an existing resource changes. |
| Resource Deleted | Fires when a resource is deleted or moved to the `deleted` status. |
| Scheduled | Fires on a recurring schedule defined by a cron expression. |

<Tip>
  Use the **Resource Updated** trigger together with a condition that checks a specific field — for example, `status = archived` — to avoid running your workflow on every minor change.
</Tip>

## Workflow Actions

When a workflow's trigger fires and its condition passes, Trofiro can execute any combination of the following actions:

- **Send email** — Deliver a formatted email to one or more recipients using a customizable template.
- **Send Slack message** — Post a message to a Slack channel or direct message via your connected Slack integration.
- **Create resource** — Automatically create a new resource in the workspace with field values you specify.
- **Update resource** — Modify one or more fields on the resource that triggered the workflow, or on another resource you identify.
- **Call webhook** — Send a signed HTTP POST request to an external URL with a JSON payload you define.
- **Run script** — Execute a JavaScript snippet in Trofiro's sandboxed runtime for custom logic that goes beyond the built-in actions.

<Note>
  Workflow execution limits vary by plan tier. The Free plan allows up to 100 workflow executions per month, the Pro plan allows 10,000, and the Business plan provides unlimited executions. If your workspace reaches its monthly limit, workflows are paused until the next billing cycle or until you upgrade. You can review your current usage under **Settings → Usage**.
</Note>
