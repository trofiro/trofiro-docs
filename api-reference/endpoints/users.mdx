The Users API lets you query and manage the members of your Trofiro workspace. Use these endpoints to list all workspace members, invite a new user, retrieve a specific user's profile, or update a user's display name and role. All user endpoints require a valid API key with at least `read:resources` scope for reads, and `write:resources` scope for writes.

---

## GET /users

Retrieve a paginated list of all members in your workspace. Results are ordered by creation date, newest first.

### Query Parameters

<ParamField query="limit" type="integer">
  Number of results to return per page. Defaults to `20`. Maximum value is `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned in `meta.next_cursor` from a previous response. Omit this parameter to start from the first page.
</ParamField>

<ParamField query="role" type="string">
  Filter results by role. Accepted values: `owner`, `admin`, `member`. Omit to return users of all roles.
</ParamField>

### Response

Returns a `data` array of user objects and a `meta` object with pagination state.

```json
{
  "data": [
    {
      "id": "usr_01H8X2K9M3",
      "name": "Jane Smith",
      "email": "jane@example.com",
      "role": "admin",
      "created_at": "2024-01-10T08:00:00Z"
    }
  ],
  "meta": {
    "has_more": false,
    "next_cursor": null
  }
}
```

<ResponseField name="data" type="array">
  Array of user objects.

  <ResponseField name="data[].id" type="string">
    Unique identifier for the user, prefixed with `usr_`.
  </ResponseField>

  <ResponseField name="data[].name" type="string">
    Display name of the user.
  </ResponseField>

  <ResponseField name="data[].email" type="string">
    Email address associated with the user's account.
  </ResponseField>

  <ResponseField name="data[].role" type="string">
    The user's role in the workspace. One of `owner`, `admin`, or `member`.
  </ResponseField>

  <ResponseField name="data[].created_at" type="string">
    ISO 8601 timestamp indicating when the user joined the workspace.
  </ResponseField>
</ResponseField>

<ResponseField name="meta.has_more" type="boolean">
  `true` if additional pages of results exist.
</ResponseField>

<ResponseField name="meta.next_cursor" type="string | null">
  Cursor to pass in the next request. `null` when you have reached the last page.
</ResponseField>

## POST /users

Invite a new member to your Trofiro workspace. The invited user receives an email prompting them to accept the invitation. Requires `write:resources` scope.

### Body Parameters

<ParamField body="email" type="string" required>
  Email address of the person to invite. If an account already exists with this address, the user is added to the workspace directly; otherwise, an invitation email is sent.
</ParamField>

<ParamField body="name" type="string">
  Display name for the new user. If omitted, the name is taken from the user's existing Trofiro account or set to the email address prefix until they update it.
</ParamField>

<ParamField body="role" type="string">
  Role to assign to the new user. Accepted values: `admin`, `member`. Defaults to `member`. Only users with the `owner` or `admin` role can assign the `admin` role.
</ParamField>

### Example

**Request:**

```bash
curl -X POST https://api.trofiro.com/v1/users \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "alex@example.com",
    "name": "Alex Johnson",
    "role": "member"
  }'
```

**Response:**

```json
{
  "data": {
    "id": "usr_02J9Y3L0N4",
    "name": "Alex Johnson",
    "email": "alex@example.com",
    "role": "member",
    "created_at": "2024-01-15T10:30:00Z"
  },
  "meta": {
    "request_id": "req_ghi789",
    "timestamp": "2024-01-15T10:30:00Z"
  }
}
```

## GET /users/:id

Retrieve a single workspace member by their user ID.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user. User IDs are prefixed with `usr_` and can be found in the `GET /users` response.
</ParamField>

### Response

Returns a single user object inside `data`.

<ResponseField name="data.id" type="string">
  Unique identifier for the user.
</ResponseField>

<ResponseField name="data.name" type="string">
  Display name of the user.
</ResponseField>

<ResponseField name="data.email" type="string">
  Email address associated with the user's account.
</ResponseField>

<ResponseField name="data.role" type="string">
  The user's workspace role: `owner`, `admin`, or `member`.
</ResponseField>

<ResponseField name="data.created_at" type="string">
  ISO 8601 timestamp indicating when the user joined the workspace.
</ResponseField>

**Example request:**

```bash
curl https://api.trofiro.com/v1/users/usr_01H8X2K9M3 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Example response:**

```json
{
  "data": {
    "id": "usr_01H8X2K9M3",
    "name": "Jane Smith",
    "email": "jane@example.com",
    "role": "admin",
    "created_at": "2024-01-10T08:00:00Z"
  },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-15T10:30:00Z"
  }
}
```

## PATCH /users/:id

Update a workspace member's display name or role. Only fields included in the request body are modified — omitted fields retain their current values.

<Note>
  Only users with the `owner` or `admin` role can update another user's role. A `member` can update their own `name` but cannot change roles. Attempting to update a role without sufficient privileges returns `403 Forbidden`.
</Note>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to update.
</ParamField>

### Body Parameters

<ParamField body="name" type="string">
  Updated display name for the user.
</ParamField>

<ParamField body="role" type="string">
  Updated role for the user. Accepted values: `owner`, `admin`, `member`. Requires the requesting key to belong to an owner or admin.
</ParamField>

### Example

**Request:**

```bash
curl -X PATCH https://api.trofiro.com/v1/users/usr_01H8X2K9M3 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Smith-Williams",
    "role": "member"
  }'
```

**Response:**

```json
{
  "data": {
    "id": "usr_01H8X2K9M3",
    "name": "Jane Smith-Williams",
    "email": "jane@example.com",
    "role": "member",
    "created_at": "2024-01-10T08:00:00Z"
  },
  "meta": {
    "request_id": "req_def456",
    "timestamp": "2024-01-15T11:00:00Z"
  }
}
```
