Webhooks allow your application to receive real-time HTTP notifications when events occur in your Trofiro workspace. Instead of polling the API repeatedly, you register a URL that Trofiro calls automatically whenever a subscribed event fires — such as when a resource is created or a workflow completes. This makes webhooks the right choice for building event-driven integrations, automation triggers, and live sync pipelines.

## Webhook Events

Subscribe to one or more of the following event types when creating a webhook endpoint:

| Event | Description |
|---|---|
| `resource.created` | A new resource was created in the workspace |
| `resource.updated` | One or more fields on a resource were changed |
| `resource.deleted` | A resource was permanently deleted |
| `workflow.triggered` | A workflow was triggered manually or by an automation rule |
| `workflow.completed` | A workflow finished running, whether successfully or with errors |

## GET /webhooks

Retrieve a list of all webhook endpoints configured in your workspace.

### Response

Returns a `data` array of webhook objects. Each object has the same shape as the `POST /webhooks` response.

**Example request:**

```bash
curl https://api.trofiro.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Example response:**

```json
{
  "data": [
    {
      "id": "wh_01H8X2K9M3",
      "url": "https://your-app.example.com/webhooks/trofiro",
      "events": ["resource.created", "resource.updated"],
      "secret": "whsec_my_signing_secret",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "meta": {
    "has_more": false,
    "next_cursor": null
  }
}
```

## POST /webhooks

Register a new webhook endpoint. Trofiro will send HTTP `POST` requests to the URL you provide whenever any of the specified events occur.

### Body Parameters

<ParamField body="url" type="string" required>
  The destination URL that Trofiro will call when an event fires. Must be a valid HTTPS URL — plain HTTP endpoints are not accepted.
</ParamField>

<ParamField body="events" type="array" required>
  An array of event type strings to subscribe to. Must include at least one event. See [Webhook Events](#webhook-events) for valid values.
</ParamField>

<ParamField body="secret" type="string">
  An optional signing secret used to verify that payloads originate from Trofiro. If provided, Trofiro includes an `X-Trofiro-Signature` header on every delivery. See [Verifying Webhook Signatures](#verifying-webhook-signatures).
</ParamField>

### Example

**Request:**

```bash
curl -X POST https://api.trofiro.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.example.com/webhooks/trofiro",
    "events": ["resource.created", "resource.updated"],
    "secret": "whsec_my_signing_secret"
  }'
```

**Response:**

```json
{
  "data": {
    "id": "wh_01H8X2K9M3",
    "url": "https://your-app.example.com/webhooks/trofiro",
    "events": ["resource.created", "resource.updated"],
    "secret": "whsec_my_signing_secret",
    "created_at": "2024-01-15T10:30:00Z"
  },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-15T10:30:00Z"
  }
}
```

## DELETE /webhooks/:id

Remove a webhook endpoint from your workspace. Trofiro will immediately stop delivering events to the associated URL.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the webhook to delete. Webhook IDs are prefixed with `wh_`.
</ParamField>

### Response

Returns `204 No Content` on success with an empty response body.

```bash
curl -X DELETE https://api.trofiro.com/v1/webhooks/wh_01H8X2K9M3 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Verifying Webhook Signatures

When you provide a `secret` when creating a webhook, Trofiro signs every payload with **HMAC-SHA256** and includes the resulting hex digest in the `X-Trofiro-Signature` request header. Verifying this signature on your server confirms that the delivery came from Trofiro and that the payload has not been tampered with.

The signature is computed over the raw JSON-serialized payload body using your signing secret. The following Node.js example shows how to verify a delivery:

```javascript
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expectedSig = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSig)
  );
}
```

Call this function inside your webhook handler, passing the parsed request body, the value of the `X-Trofiro-Signature` header, and your webhook secret. Reject any request for which the function returns `false` — it means the payload is invalid or not from Trofiro.

<Warning>
  Use `crypto.timingSafeEqual` (or your language's equivalent) to compare signatures. A standard string equality check is vulnerable to timing attacks, which could allow an attacker to forge a valid signature.
</Warning>

## Webhook Payload Format

Every webhook delivery is an HTTP `POST` request with a JSON body. The payload follows a consistent envelope format regardless of the event type:

```json
{
  "event": "resource.created",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "id": "res_01H8X2K9M3",
    "name": "Q1 Planning",
    "status": "active",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `event` | string | The event type that triggered the delivery (e.g., `resource.created`) |
| `timestamp` | string | ISO 8601 timestamp of when the event occurred |
| `data` | object | The affected resource or workflow object at the time of the event |

Trofiro expects your endpoint to respond with a `2xx` status code within **10 seconds**. If your endpoint times out or returns a non-2xx response, Trofiro will retry the delivery up to **3 times** with exponential backoff.

<Tip>
  Use a tool like [webhook.site](https://webhook.site) to inspect real webhook payloads during development before pointing your live endpoint at Trofiro. It gives you a temporary public URL that logs every incoming request in full, making it easy to validate your payload handling before going live.
</Tip>
