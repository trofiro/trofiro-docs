Integrations let Trofiro participate in the workflows you already have rather than forcing you to leave the tools your team depends on. When an event happens in an external service — a pull request, a message, a trigger in an automation platform — Trofiro can react to it automatically, and vice versa. To browse everything available, go to **Settings > Integrations**.

<Note>
  Only users with the **Admin** or **Owner** role can add or remove integrations. Members and Viewers can use integrations that have already been configured but cannot change the connection settings.
</Note>

## Browse Available Integrations

Navigate to **Settings > Integrations** to open the integration catalog. Each tile shows the integration name, a brief description, and its current connection status. Use the search bar at the top of the catalog to find a specific tool, or browse by category using the filter tabs.

## Setting Up Slack

<Steps>
  <Step title="Open the Slack integration page">
    Go to **Settings > Integrations > Slack**.
  </Step>
  <Step title="Connect your Slack workspace">
    Click **Connect Slack**. You are redirected to Slack's authorization page.
  </Step>
  <Step title="Authorize Trofiro">
    Review the permissions Trofiro requests and click **Allow** to authorize the connection. You are returned to Trofiro automatically once authorization is complete.
  </Step>
  <Step title="Choose a default notification channel">
    In the Slack integration settings, select the Slack channel where Trofiro should send notifications by default. You can override this channel for individual notification types in **Settings > Notifications**.
  </Step>
  <Step title="Save your settings">
    Click **Save**. Trofiro sends a confirmation message to the selected channel to confirm the connection is active.
  </Step>
</Steps>

## Setting Up GitHub

To connect GitHub, go to **Settings > Integrations > GitHub** and click **Connect GitHub**. You are prompted to authorize Trofiro via GitHub's OAuth flow.

After authorization, configure the integration:

1. **Select repositories** — choose which GitHub repositories Trofiro should monitor. You can select individual repos or grant access to all repositories in an organization.
2. **Choose tracked events** — decide which GitHub events should create or update Trofiro resources. Available events include pull request opened, pull request merged, issue created, and issue closed.
3. Click **Save** to activate the connection.

Once active, Trofiro creates a new resource for each tracked event and links it back to the originating GitHub object.

## Using Zapier

Trofiro publishes an official app in the Zapier integration marketplace, allowing you to connect Trofiro to thousands of other services without writing any code. Visit the [Trofiro app on Zapier](https://zapier.com/apps/trofiro) to browse ready-made Zap templates or build your own.

**Triggers** — events in Trofiro that can start a Zap:
- Resource Created
- Resource Updated
- Resource Archived
- Workflow Triggered

**Actions** — operations Zapier can perform in Trofiro on behalf of another app:
- Create Resource
- Update Resource
- Add Tag to Resource
- Trigger Workflow

To set up a Zap, authenticate Trofiro inside Zapier using an API key from **Settings > API**. Zapier guides you through the rest of the configuration.

## Custom Webhooks

For custom server-to-server integrations, Trofiro supports outbound webhooks that deliver a signed HTTP POST payload to any URL you control whenever a specified event occurs. Webhooks are a flexible alternative when a pre-built integration does not exist for your tool.

For full details on registering endpoints, verifying signatures, and handling delivery retries, see the [Webhooks API reference](/api-reference/endpoints/webhooks).
