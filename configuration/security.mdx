Trofiro provides multiple layers of security to protect your account and your team's data. You can enable two-factor authentication for individual accounts, configure single sign-on for your organization, manage API keys with fine-grained control, and review active sessions — all from the **Settings > Security** panel.

## Two-Factor Authentication (2FA)

Two-factor authentication adds a second verification step to your login process, significantly reducing the risk of unauthorized access even if your password is compromised. Trofiro uses time-based one-time passwords (TOTP), compatible with any standard authenticator app.

<Steps>
  <Step title="Open the 2FA settings">
    Navigate to **Settings > Security > Two-Factor Authentication**.
  </Step>
  <Step title="Enable 2FA">
    Click **Enable 2FA**. Trofiro will generate a QR code and a setup key.
  </Step>
  <Step title="Scan the QR code">
    Open your authenticator app — such as **Google Authenticator**, **Authy**, or **1Password** — and scan the QR code displayed on screen. If you cannot scan the code, enter the setup key manually.
  </Step>
  <Step title="Confirm the 6-digit code">
    Enter the 6-digit code currently shown in your authenticator app and click **Verify**. This confirms that your app is correctly synced with Trofiro.
  </Step>
  <Step title="Save your backup codes">
    Trofiro will display a set of one-time backup codes. Copy these codes and store them in a secure location (such as a password manager). Each backup code can be used once to access your account if you lose access to your authenticator app.
  </Step>
</Steps>

Once 2FA is enabled, you will be prompted to enter a code from your authenticator app each time you log in. Workspace admins can enforce 2FA for all members at **Settings > Security > Policies**.

## Single Sign-On (SSO)

<Info>
  SSO is available on the **Enterprise plan** only. Contact [support@trofiro.com](mailto:support@trofiro.com) to learn more about upgrading.
</Info>

Single sign-on lets your team authenticate with Trofiro using your organization's existing identity provider, eliminating the need for separate Trofiro passwords and centralizing access control.

Trofiro supports the following SSO providers:

- **Google Workspace** — authenticate via your organization's Google account.
- **Microsoft Entra ID (Azure AD)** — connect using Microsoft's identity platform.
- **Okta** — integrate with Okta's identity and access management service.
- **Auth0** — use Auth0 as your identity broker for custom or federated identity setups.

A workspace **admin** must configure SSO on behalf of the entire organization. To set it up, go to **Settings > Security > SSO**, select your identity provider, and follow the provider-specific configuration steps to enter your client ID, client secret, and metadata URL. Once SSO is active, members can sign in via the **Continue with SSO** button on the Trofiro login page using their organization email address.

## API Key Management

API keys authenticate requests to the Trofiro REST API. Manage all of your workspace's API keys at **Settings > API Keys**.

<Warning>
  **Never commit API keys to source control or share them in plain text** — in chat, email, or documentation. Treat every API key like a password. If you suspect a key has been exposed, revoke it immediately and generate a replacement. Use environment variables or a secrets manager to inject keys into your applications at runtime.
</Warning>

### Generating a New Key

1. Go to **Settings > API Keys** and click **Generate New Key**.
2. Enter a descriptive name for the key that identifies its purpose (e.g., `ci-pipeline-prod` or `zapier-integration`).
3. Optionally, set an **expiry date** to automatically invalidate the key after a specified period. Using expiry dates is strongly recommended for keys used in automated systems.
4. Click **Generate**. Copy the key immediately — Trofiro will only display the full key once.

### Rotating a Key

Rotating a key replaces it with a new secret while preserving the key's name and settings. This is useful for regular credential hygiene or after a suspected exposure.

1. Navigate to **Settings > API Keys** and click the name of the key you want to rotate.
2. Click **Rotate**, then confirm the action.
3. Copy the newly generated key and update any systems or integrations that use the old key. The old key is invalidated immediately upon rotation.

### Revoking a Key

To permanently invalidate a key, click **Revoke** next to the key in the list and confirm. Revoked keys cannot be restored — any system using that key will receive `401 Unauthorized` errors until a new key is configured.

## Active Sessions

Trofiro tracks every active session tied to your account so you can identify and terminate any sessions you do not recognize.

To review your sessions, go to **Settings > Security > Sessions**. Each session entry shows:

- The device type and browser
- The approximate location based on IP address
- The date and time of the last activity

To revoke a specific session, click **Revoke** next to it. To sign out of all sessions simultaneously — for example, if you believe your account has been compromised — click **Revoke All Sessions**. This will immediately invalidate all active sessions and require a fresh login on every device.

<Note>
  **Audit logs** — a full record of actions taken within your workspace, including logins, permission changes, and resource modifications — are available on the **Pro and Enterprise plans** at **Settings > Audit Log**. Audit logs are retained for 90 days on Pro and 365 days on Enterprise.
</Note>
